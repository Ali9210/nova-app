# nova — تطبيق تواصل اجتماعي حقيقي (Backend + Frontend)

تطبيق كامل شبيه بـ X/تويتر: تسجيل حساب حقيقي (بريد إلكتروني + كلمة مرور مشفّرة)، جلسة تسجيل دخول حقيقية عبر JWT، تغريدات، إعجاب، إعادة تغريد، وتعديل الملف الشخصي مع رفع صورة حقيقي — كل ذلك محفوظ فعلياً على الخادم.

## ما بداخل المشروع
- `server.js` — خادم Express يوفّر واجهة API الحقيقية (تسجيل/دخول/تغريدات/إعجاب/رفع صورة)
- `db.js` — قاعدة بيانات بسيطة بصيغة JSON (بدون أي اعتمادية native)، لتجنّب مشاكل التثبيت على أي استضافة
- `public/index.html` — الواجهة الأمامية (React عبر CDN بدون خطوة بناء/بندلينغ)
- `data/` — يُنشأ تلقائياً، يحوي `users.json` و `tweets.json`
- `public/uploads/` — صور البروفايل المرفوعة

## التشغيل محلياً
```bash
npm install
npm start
```
ثم افتح: http://localhost:3000

## النشر على استضافة مجانية (Render — الأسهل)
1. ارفع هذا المجلد كمستودع على GitHub.
2. ادخل إلى https://render.com → New → Web Service → اختر المستودع.
3. الإعدادات:
   - Build Command: `npm install`
   - Start Command: `npm start`
4. أضف متغيّر بيئة (Environment Variable):
   - `JWT_SECRET` = أي نص عشوائي طويل وسرّي (مهم جداً، لا تتركه فارغاً في الإنتاج)
5. اضغط Deploy — بعد دقائق تحصل على رابط عام مثل:
   `https://nova-yourname.onrender.com`
   هذا هو رابط تطبيقك الحقيقي المباشر — أي شخص يفتحه يقدر يسجّل حساباً ويستخدم التطبيق.

### بديل: Railway
نفس الخطوات تقريباً على https://railway.app — أنشئ مشروعاً من GitHub، أضف `JWT_SECRET`، وسيُشغَّل تلقائياً عبر `npm start`.

⚠️ ملاحظة عن التخزين: هذا المشروع يحفظ البيانات في ملفات JSON على القرص. هذا يعمل تماماً للتجربة والاستخدام الشخصي، لكن على بعض الاستضافات المجانية (مثل Render Free) قد يُعاد تشغيل القرص من الصفر بعد فترة خمول طويلة، فتُفقد البيانات. للاستخدام الجاد على المدى الطويل، اربط قاعدة بيانات حقيقية (مثل Postgres المجاني من Render أو Railway) — إذا احتجت مساعدة بهذا التحويل أخبرني.

## تفعيل تسجيل الدخول عبر جوجل (اختياري)
1. اذهب إلى https://console.cloud.google.com → أنشئ مشروعاً.
2. من "APIs & Services" → "Credentials" → أنشئ "OAuth Client ID" من نوع "Web application".
3. أضف رابط تطبيقك (مثل `https://nova-yourname.onrender.com`) ضمن Authorized JavaScript origins.
4. انسخ الـ Client ID، وأضفه كمتغيّر بيئة على استضافتك:
   - `GOOGLE_CLIENT_ID` = القيمة التي نسختها
5. أعد النشر — سيظهر زر "تسجيل الدخول عبر جوجل" تلقائياً في صفحة الدخول.

بدون هذه الخطوة، التطبيق يعمل بشكل كامل عبر البريد الإلكتروني وكلمة المرور فقط.

## أمان يجب الانتباه له قبل الاستخدام الجاد
- غيّر `JWT_SECRET` دائماً في الإنتاج (لا تستخدم القيمة الافتراضية).
- كلمات المرور مشفّرة عبر bcrypt (وليست نصاً عادياً).
- لا يوجد حالياً تحقق من البريد الإلكتروني (email verification) ولا استرجاع كلمة مرور منسية — يمكن إضافتهما لاحقاً عند الحاجة.
- الحد الأقصى لحجم الصورة المرفوعة 4MB.

## هيكل واجهة API
| Method | Path | الوصف |
|---|---|---|
| POST | /api/auth/register | إنشاء حساب |
| POST | /api/auth/login | تسجيل الدخول |
| POST | /api/auth/google | تسجيل الدخول عبر جوجل |
| GET | /api/auth/me | بيانات المستخدم الحالي |
| PUT | /api/auth/me | تعديل الاسم/اليوزر/النبذة |
| POST | /api/auth/avatar | رفع صورة الملف الشخصي |
| GET | /api/tweets | كل التغريدات |
| POST | /api/tweets | نشر تغريدة |
| POST | /api/tweets/:id/like | إعجاب/إلغاء إعجاب |
| POST | /api/tweets/:id/retweet | إعادة تغريد/إلغاء |
